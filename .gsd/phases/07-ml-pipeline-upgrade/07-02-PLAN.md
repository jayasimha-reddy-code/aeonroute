  **Environment reward shaping (src/environment.py):**
  In the EVRoutingEnvironment._calculate_reward() method (around line 339-399), add potential-based shaping:

  1. Define a potential function based on distance to goal:
  `python
  def _potential(self, node):
      """Distance-based potential: closer to goal = higher potential."""
      if self.destination is None:
          return 0.0
      try:
          dist = nx.shortest_path_length(self.road_graph.graph, node, self.destination, weight='distance')
          return -dist  # Negative distance = higher potential when closer
      except nx.NetworkXNoPath:
          return -1000.0  # Large negative for unreachable
  `

  2. In the step() method, compute shaping reward F = gamma * Phi(s') - Phi(s) and add it to the base reward:
  `python
  shaping_reward = 0.95 * self._potential(new_node) - self._potential(old_node)
  reward += shaping_reward * 2.0  # Scale factor for shaping influence
  `

  3. Add charging incentive: if battery_soc < 30% and the agent chooses to charge at a charging station, give bonus +10. Currently only penalties exist for bad charging decisions (line ~380).

  4. Cache the shortest path distances to avoid recomputing potential every step (use a dict self._potential_cache).

  **Q-Learning convergence monitoring (src/q_learning_agent.py):**
  Add a rolling average tracker to QLearningAgent:

  `python
  def __init__(self, ...):
      ...
      self.reward_history = []
      self.convergence_window = 50
      self.convergence_threshold = 5.0  # reward change threshold

  def update_convergence(self, episode_reward):
      self.reward_history.append(episode_reward)
      if len(self.reward_history) >= self.convergence_window * 2:
          recent = np.mean(self.reward_history[-self.convergence_window:])
          previous = np.mean(self.reward_history[-2*self.convergence_window:-self.convergence_window])
          self.converged = abs(recent - previous) < self.convergence_threshold
  `

  Add a converged property that training loops can check. Do NOT change the core Q-learning update equation (Bellman update) — only add monitoring on top.
