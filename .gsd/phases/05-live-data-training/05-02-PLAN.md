  <action>
    **1. Create `frontend/src/hooks/useSSE.ts`** -- reusable EventSource hook:
    - Accept options: { url, onMessage, onError?, onOpen?, enabled?, maxRetries?, baseDelay?, maxDelay? }
    - Use useRef for EventSource instance (NOT useState -- avoids re-render on reconnect)
    - On open: reset retry counter, call onOpen
    - On message: call onMessage (generic handler)
    - Also add named event listeners for 'progress', 'complete', 'stopped', 'idle'
      (matching the backend named SSE events via es.addEventListener)
    - On error: close EventSource, exponential backoff reconnect
      delay = Math.min(baseDelay * 2^retries, maxDelay) with defaults 1000ms base, 30000ms max
    - If retries exceed maxRetries (default 10), stop reconnecting
    - Cleanup: close EventSource and clear timeout on unmount via useEffect return
    - Return { close } -- connection status is pushed to Zustand (not local state)

    **2. Add training stream slice to `frontend/src/store/store.ts`**:
    - Add to SystemState interface:
      - sseConnected: boolean
      - setSSEConnected: (connected: boolean) => void
      - trainingProgress (object with is_training, progress, current_step, metrics, gan_epoch, gan_total_epochs, rl_episode, rl_total_episodes)
      - lossHistory: Array of { epoch, g_loss, d_loss_real, d_loss_fake }
      - rewardHistory: Array of { episode, reward, length }
      - updateTrainingFromSSE: (data: any) => void
      - resetTrainingData: () => void
    - Implement updateTrainingFromSSE:
      - Merge incoming status fields into trainingProgress
      - If data.new_loss_points is non-null array, append each point to lossHistory
      - If data.new_reward_points is non-null array, append each point to rewardHistory
      - Cap lossHistory at 1000 entries and rewardHistory at 2000 entries
    - Implement resetTrainingData: clear lossHistory, rewardHistory, reset trainingProgress
    - Add selector hooks: useTrainingProgress, useLossHistory, useRewardHistory, useSSEConnected
    - Do NOT persist training data (exclude from partialize)

    **3. Create `frontend/src/hooks/useTrainingStream.ts`** -- bridge hook:
    - Accept enabled: boolean parameter
    - Get updateTrainingFromSSE and setSSEConnected from Zustand (stable refs)
    - Create stable onMessage callback that parses event.data as JSON and calls updateTrainingFromSSE
    - Create stable onOpen/onError callbacks that set SSE connection state
    - Pass all to useSSE hook with URL = VITE_API_URL + '/api/training/stream'
    - Export the hook for use in Training.tsx
  </action>
