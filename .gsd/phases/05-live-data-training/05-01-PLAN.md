<objective>
Enrich the backend ML pipeline with per-epoch/episode callback mechanisms so that GAN loss
and Q-Learning reward data stream through SSE to the frontend in real-time.

Purpose: The frontend needs live loss/reward data to render animated charts during training.
Currently, the SSE endpoint only sends basic progress — no per-iteration metrics. This plan
closes that gap by adding callbacks to the ML training loops, wiring them through
TrainingService into AppState, enriching the SSE generator to include delta data, and adding
a temporal traffic endpoint for the time-of-day slider.

Output: Backend capable of streaming per-epoch GAN loss + per-episode RL reward via SSE,
plus a new /api/traffic-patterns/temporal endpoint.
</objective>
