<task type="auto">
  <name>Task 1: Playwright visual regression config and npm scripts</name>
  <files>frontend/playwright.config.ts, frontend/package.json</files>
  <action>
  Update Playwright configuration for visual regression testing.

  In frontend/playwright.config.ts:
  1. Add `expect` config with `toHaveScreenshot` settings: `maxDiffPixelRatio: 0.01`, `threshold: 0.2`
  2. Set `viewport: { width: 1280, height: 720 }` in `use` block for consistent screenshots
  3. Keep existing settings (headless, retries, webServer)
  4. Add `snapshotPathTemplate: '{testDir}/__screenshots__/{testFilePath}/{arg}{ext}'` for organized snapshot storage
  5. Ensure `projects` array has at least chromium for consistent rendering

  In frontend/package.json scripts:
  1. Add `"test:e2e": "playwright test"`
  2. Add `"test:e2e:update": "playwright test --update-snapshots"`
  3. Keep existing scripts unchanged
  </action>
  <verify>Run `cd frontend && npx playwright test --list` — lists all test files including visual regression</verify>
  <done>Playwright config optimized for visual regression. npm scripts added for e2e and snapshot update.</done>
</task>

<task type="auto">
  <name>Task 2: Visual regression specs for key pages</name>
  <files>frontend/e2e/visual-regression.spec.ts</files>
  <action>
  Create visual regression test file covering 4 key application pages.

  Create frontend/e2e/visual-regression.spec.ts:
  1. Import `test, expect` from `@playwright/test`
  2. Add `test.describe('Visual Regression')` block
  3. Add `test.beforeEach` that navigates to base URL and waits for app to load (wait for `main` element)

  Test cases (each navigates and captures full-page screenshot):
  1. Dashboard page: navigate to '/', wait for stat cards to render, `await expect(page).toHaveScreenshot('dashboard.png', { fullPage: true })`
  2. Route Planner page: navigate to '/route-planner', wait for map container, `await expect(page).toHaveScreenshot('route-planner.png', { fullPage: true })`
  3. Training page: navigate to '/training', wait for training controls, `await expect(page).toHaveScreenshot('training.png', { fullPage: true })`
  4. Analytics page: navigate to '/analytics', wait for charts or "no data" message, `await expect(page).toHaveScreenshot('analytics.png', { fullPage: true })`

  For each page:
  - Wait for network idle or specific element visibility before screenshot
  - Use `page.waitForLoadState('networkidle')` with a timeout fallback
  - Add 500ms `page.waitForTimeout` after load for animations to settle

  Add a dark mode visual regression test:
  5. Dashboard dark mode: toggle theme via UI (click theme button), wait, screenshot 'dashboard-dark.png'

  Note: First run will ALWAYS fail since no baselines exist. That's expected — run `npx playwright test --update-snapshots` to create baselines.
  </action>
  <verify>Run `cd frontend && npx playwright test e2e/visual-regression.spec.ts --update-snapshots` to generate baseline screenshots (first run creates them)</verify>
  <done>Visual regression specs capture Dashboard, Route Planner, Training, Analytics pages. Baselines generated on first --update-snapshots run.</done>
</task>
