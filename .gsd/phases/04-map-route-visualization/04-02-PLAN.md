1. **Create frontend/src/hooks/useAnimatedRoute.ts** — progressive coordinate reveal animation:
   - Hook signature: `useAnimatedRoute(fullCoords: [number, number][] | null, durationMs?: number)` returns `[number, number][]` (the visible subset of coordinates).
   - Default durationMs = 2000 (2 seconds).
   - Use requestAnimationFrame loop:
     - On each frame, compute `progress = Math.min((now - startTime) / durationMs, 1)`.
     - Compute `count = Math.max(2, Math.ceil(progress * fullCoords.length))` — minimum 2 points for a valid line.
     - Set visible coords to `fullCoords.slice(0, count)`.
     - Continue until progress reaches 1.
   - Use a ref for the animation frame ID to cancel on cleanup.
   - Reset animation when fullCoords reference changes (new route selected/generated).
   - Return empty array if fullCoords is null.
   - Cancel requestAnimationFrame in useEffect cleanup to prevent stale closures.
   - **No turf.js dependency** — uses the simpler coordinate-slicing approach from research.

2. **Create frontend/src/components/map/RouteLayer.tsx** — route visualization component:
   - Props interface:
     - routes: Route[] (all routes to display)
     - highlightIndex: number | undefined (which route is selected)
     - posLookup: Record<string, [number, number]> | null (nodeId to [lng,lat])
     - animate?: boolean (default true, enables progressive reveal on highlighted route)
   - Import Source, Layer from react-map-gl/maplibre.
   - Import routeToGeoJSON from lib/geo.ts.
   - Import useAnimatedRoute from hooks.
   - Import ROUTE_COLORS from mapStyles.

   **Non-highlighted routes (solid color, muted):**
   - For each route where idx !== highlightIndex (up to 3 routes), render:
     - Source with id=`route-alt-{idx}`, type="geojson", data from routeToGeoJSON.
     - Layer id=`route-alt-line-{idx}`, type="line":
       - line-color: ROUTE_COLORS[idx]
       - line-width: 2.5
       - line-opacity: 0.35
       - line-dasharray: [6, 4] (dashed for non-selected)
     - Layout: line-cap 'round', line-join 'round'.

   **Highlighted route (energy gradient + animation):**
   - Extract the highlighted route: `routes[highlightIndex]`.
   - Compute full coordinates from routeToGeoJSON.
   - Pass full coordinates to useAnimatedRoute hook to get animated visible coords.
   - Build animated GeoJSON Feature from visible coords.
   - Render:
     - **Glow underlay Source + Layer** (id=`route-highlight-glow`):
       - Source: type="geojson", data=animatedGeoJSON (animated coords).
       - Layer: type="line", line-color: ROUTE_COLORS[highlightIndex], line-width: 10, line-opacity: 0.15, line-blur: 4.
     - **Energy gradient Source + Layer** (id=`route-highlight-energy`):
       - Source: type="geojson", data=animatedGeoJSON, **lineMetrics={true}** (critical for line-gradient).
       - Layer: type="line", paint:
         - line-width: 5
         - line-gradient: ['interpolate', ['linear'], ['line-progress'], 0, '#10b981', 0.4, '#f59e0b', 0.8, '#ef4444', 1, '#dc2626']
         - Note: line-gradient is incompatible with line-dasharray, so do NOT set dasharray on the gradient layer.
       - Layout: line-cap 'round', line-join 'round'.

   **Key details:**
   - Memoize GeoJSON objects with useMemo to avoid unnecessary recomputation.
   - Use stable Source/Layer IDs — do NOT use route data in the ID, use index position only.
   - If no routes or posLookup is null, return null.
   - If highlightIndex is undefined, default to 0 (first route highlighted).
   - When animate is false, skip useAnimatedRoute and show full coordinates immediately.
