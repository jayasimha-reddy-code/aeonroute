<task type="auto" id="14-02-T2">
  <name>Write real Q-Learning training loop</name>
  <files>backend/app/services/training_service.py (new or rewrite)</files>
  <action>
    1. Create/rewrite `backend/app/services/training_service.py` with a `run_q_learning_training()` function that:
       a. Loads the Hyderabad graph from `state.hyderabad_graph`
       b. Initializes Q-table as `dict[(current_node,), np.array]` for each node
       c. Runs `MAX_TRAINING_EPISODES` episodes of epsilon-greedy Q-learning
       d. Each episode: random source/dest, traverse graph edges via Q-values, update Q(s,a) with Bellman equation
       e. Reward shaping: +100 for reaching dest, -1 per step, +10 for using charging station, -50 for dead end
       f. After each episode, yield SSE events for streaming
       g. After training completes, save Q-table to `models/q_learning/q_table_hyderabad.pkl`
       h. Store Q-table in `state.q_table` so routing service can use it immediately
    2. Respect `settings.MAX_TRAINING_EPISODES` from `.env` / Pydantic Settings
    3. Support cancellation via a threading event

    **ARCHITECTURAL OVERRIDE #3 — Multiplexed SSE Stream:**
    4. Every SSE yield MUST be a JSON object with a `type` discriminator field:
       a. Metric events (for charts): `{"type": "metric", "episode": n, "reward": R, "loss": td_error, "epsilon": eps}`
       b. Log events (for Wave 5 LiveTerminal): `{"type": "log", "timestamp": "ISO8601", "message": "Updating Q-table for node 142..."}`
       c. Status events: `{"type": "status", "phase": "training", "progress": 0.45}`
    5. Emit log events at key points: episode start, Q-table update, reward calculation, checkpoint save
    6. The frontend SSE handler must parse the `type` field and route:
       - `"metric"` → training chart data store
       - `"log"` → LiveTerminal log buffer
       - `"status"` → progress bar
  </action>
  <verify>

    - Q-table file is written to disk
    - Yield produces valid episode metrics
    - Q-table loaded by routing service for route generation
  </verify>
