<objective>
Add fluid, physics-based micro-interactions to every interactive surface. Pages stagger in with spring animations, cards lift with luxurious hover physics, map routes draw themselves, and nodes pulse with life. This is the final polish that makes the UI feel alive.

**Current state:**
- `motion.ts` has `staggerContainer`, `staggerItem`, `pageVariants` — good foundation
- Dashboard and training pages use these variants inconsistently
- Stations and Settings (from Wave 1) already have stagger setup
- Card has `glass-hover` with basic transition
- Map has `useAnimatedRoute` hook for progressive coordinate reveal
- No AnimatePresence wrapping routes for page transitions
- No CSS keyframes for route line draw animation
- No pulsing nodes on the map

**Target state:**
- All pages wrapped in `<motion.div variants={staggerContainer}>` with children using `staggerItem`
- Parent: `variants={{hidden: {opacity: 0}, show: {opacity: 1, transition: {staggerChildren: 0.1}}}}`
- Children: `variants={{hidden: {opacity: 0, y: 20}, show: {opacity: 1, y: 0, transition: {type: "spring", stiffness: 300, damping: 24}}}}`
- Cards: `transition-all duration-500 ease-out` with `-translate-y-1.5` lift and neon shadow bloom
- Route polylines: SVG `stroke-dasharray` + `stroke-dashoffset` CSS animation to "draw" routes
- Map nodes: `@keyframes pulse-glow` infinite animation
- `<AnimatePresence mode="wait">` wrapping `<Routes>` for cross-fade page transitions
</objective>
