# Plan 10-01: Nuke & Pave — CSS Reset + Tailwind Config + Dark Lock

## Wave 1 — Foundation (CSS/Config First)

<objective>
Destroy the conflicting CSS variable system and violet/cyan color palette that produced the washed-out grey UI. Replace with a minimal, authoritative dark-mode-only Tailwind config hardcoded to the deep midnight + neon-green + neon-gold target palette. Lock `index.html` to permanent dark mode so no theme logic can override.
</objective>

<context>
**Reference image analysis:**
The target UI is a premium dark cockpit dashboard. Key visual properties extracted from the reference:
- **Background:** Very deep navy midnight (`#0b0f19`) — nearly black with a subtle blue undertone
- **Card surfaces:** Dark, semi-opaque glass — NOT heavily transparent. More like `rgba(17, 21, 28, 0.85)` with a subtle frosted edge. Readable text sits on solid-feeling panels.
- **Card borders:** Extremely subtle (~5% white opacity), with specific cards having accent-colored border glows (gold top-border on "System Status" card)
- **Primary accent:** Neon green (`#2ecc71`) — used for active sidebar icon bg, status text ("SG-GAN Active"), progress bars, charging station icons
- **Secondary accent:** Gold/amber (`#f1c40f`) — used for route lines on map, the "System Status" card border glow at top
- **Tertiary accent:** Teal/cyan (`#1abc9c`) — used for chart lines (Energy Consumption graph)
- **Text hierarchy:** Bright white for titles/values, ~60% white for labels, ~35% white for captions
- **Sidebar:** Narrow icon-centric (~80px), icons stacked vertically with small text labels below, green ⚡ logo at top, active item has green-tinted box
- **Header:** Full-width top bar with pill-shaped search input (dark glass), notification bell with red dot, "SG-GAN EV Routing" brand+logo on far right
- **Cards:** Have `...` overflow menu dots in top-right corner, rounded corners (~16px), no heavy frosted blur — more like solid dark panels with subtle glass edge

**Why Phase 09 failed:** `:root` defined 60+ violet/lavender tokens, `.dark` overrode ~40, and components used a mix of CSS vars, Tailwind arbitrary values, and Tailwind semantic classes. Three competing color systems → everything averaged to grey.

**Nuclear approach:** Delete ALL CSS custom properties except layout/motion tokens. Move ALL color authority into `tailwind.config.ts`. Hardcode permanent dark mode. Remove theme-cycling.
</context>

<dependencies>
None — this is the foundation. Must execute before Plans 10-02 and 10-03.
</dependencies>

<task id="1">
  <name>Nuke index.css — strip all color/glass custom properties</name>
  <files>frontend/src/index.css</files>
  <action>
    **Delete** the entire `:root { ... }` block and the entire `.dark { ... }` block from index.css. These contain ~100 CSS custom properties that conflict with Tailwind.

    **Keep only:**
    - `@tailwind base/components/utilities` directives
    - Layout tokens: `--sidebar-width`, `--sidebar-collapsed`, `--header-height`
    - Motion tokens: `--ease-out-expo`, `--duration-*`, `--motion-*`, `--spring-*`
    - Radius tokens: `--radius-*`
    - MapLibre-specific overrides (`.maplibregl-*` blocks)
    - `@keyframes` animation definitions
    - Reduced-motion media query (`@media (prefers-reduced-motion)`)
    - Presentation mode overrides (`.presentation` class)

    **Delete all of these:**
    - ALL `--color-*` variables
    - ALL `--glass-*` variables  
    - ALL `--neon-*` variables
    - ALL `--text-*` typography color variables (keep font size scale)
    - ALL `--focus-*` and `--glow-*` variables
    - ALL `--surface-*` variables
    - ALL component classes that use CSS vars for colors: `.card`, `.card-glass`, `.card-interactive`, `.card-glow`, `.btn-primary`, `.btn-secondary`, `.btn-ghost`, `.btn-danger`, `.input-field`, `.badge-*`, `.glass`, `.glass-elevated`, `.glass-glow-*`
    - The `.dark` override block entirely

    ```css
    @layer base {
      html {
        -webkit-font-smoothing: antialiased;
        -moz-osx-font-smoothing: grayscale;
      }
      body {
        @apply bg-background text-white font-sans;
      }
      * {
        @apply border-white/5;
      }
    }
    ```

    The resulting index.css should be under 300 lines (down from 1,449).
  </action>

    **Why `0.85` opacity, not `0.5`:** The reference image cards are clearly readable dark panels that feel solid. They are NOT deeply transparent frosted glass — you cannot see content behind them. The subtle glass effect comes from the barely-visible border and backdrop-blur on the edge, not from transparency.

    **Why `neon-gold` not `neon-yellow`:** The reference uses a warm gold (#f1c40f), especially visible in the "System Status" card's amber top border glow and the yellow route lines on the map. "Gold" better describes this hue.

    **New: `glow-gold-top` shadow:** The reference "System Status" card has a distinct golden glow along its top edge. This is achieved with `inset 0 1px 0 0 rgba(241, 196, 15, 0.4)` — a horizontal inset shadow at the top.

    **New: `surface-inset`:** The reference shows rows within cards (like model status rows) sitting in darker inset areas. This `rgba(0, 0, 0, 0.25)` provides that recessed effect.
